踏上GIT之路-1

2022-12-11

前言

GIT 在我目前認知若簡而言之的話，就是一種用來備份 vscode，
較專業且體積小速度快方便追朔過往紀錄，但要靈活運用可要先吃點苦頭呢～
所以記住老師的話，在 Git 初學階段每天都要用才不會忘記！
或透過線上小遊戲突破狀況題來鍛鍊鍛鍊。
若有闡述錯誤也歡迎大家不吝嗇鞭策、糾正我這個新手上路的！

Git 語法

用 vscode 創一個 git 資料夾初始化
避免資料夾層次過多的版控

  • Init 資料夾初始化用 git init

  • Status 看狀態的概念 git status
    通常 add commit 是一個好兄弟作用，add 完所需就要拍照 commit 紀錄

  • Add 新增檔案後用 git add 檔案名.html（型態）
    Add 會先在暫存區

  • Commit - m “” 拍照紀錄並留言為了什麼紀錄留訊息
    例如
    git commit -m “#23 ”
    git commit -m “add cat1.html”
    commit 後面還有一大段延伸分支用小心搞混（可能只有我會搞混＠＠）

  • Log 可查看超多紀錄 git log （大量訊息偏細節）
    也可以像這樣 git log —oneline（顯示每個座標 commit 訊息）如下圖

  • Checkout 轉移 HEAD（可追朔檔案 時光機概念）

Git 分支的出現

同一個檔案進度更新就給他存檔（add）拍照（commit-m）起來。
至於進度更新就會產出 C0~C2 的更新，示意圖如下。標配一張貼紙 main


<圖 1>

我們可以去追朔檔案歷史長相，甚是把他們想像是平行時空，一種東西有不同選擇造成不同結果版本的意思～可以同時存在多種的檔案，甚至可以彙整合併成產出最終定案版本。

如下<圖 2>，C1 後面延伸出岔路就是平行時空，利用版本 C1 做了兩種選擇（更改）版本並各自 commit 的意思。

![](https://i.imgur.com/qDhZGD4.jpg =200x) ![](https://i.imgur.com/qDhZGD4.jpg =200x)

換一種方式說的話
完成一個段落後從 C1 變成 commit C2
若當我又去追朔了 C1 時期且更動後存一個 commit 成 C3
就成了下面這張圖，產生分支！


<圖 2>

每一條路都要有個貼紙指示著，如同前面說的是標配
這時候出現兩種像是貼紙的標籤。<圖 2>
在沒有貼紙的道路上，等於默許它引退（下圖 C3 就會消失，不是刪除）


<圖 3>
